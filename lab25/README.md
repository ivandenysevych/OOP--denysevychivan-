# Лабораторна робота №25: Інтеграція патернів проєктування

## Мета роботи
Розробка комплексної системи, що демонструє взаємодію чотирьох фундаментальних патернів: **Singleton, Factory Method, Strategy та Observer**.

## Опис реалізованих патернів

### 1. Singleton (Одинак)
Реалізований у класі LoggerManager. Він гарантує, що в усій системі існує лише один об'єкт для керування логуванням, забезпечуючи глобальну точку доступу до нього.

### 2. Factory Method (Фабричний метод)
Абстрактний клас LoggerFactory та його реалізації (ConsoleLoggerFactory, FileLoggerFactory) дозволяють створювати об'єкти логерів, не прив'язуючись до конкретних класів. Це дає змогу динамічно змінювати спосіб виводу даних.

### 3. Strategy (Стратегія)
Інтерфейс IDataProcessorStrategy дозволяє змінювати алгоритм обробки даних (шифрування або стиснення) "на льоту". Клас DataContext делегує виконання операції обраній стратегії.

### 4. Observer (Спостерігач)
Реалізований через події (event Action). Клас DataPublisher сповіщає всіх підписаних спостерігачів (зокрема ProcessingLoggerObserver) про завершення обробки даних.



## Демонстрація сценаріїв
1.  **Повна інтеграція:** Логування в консоль оброблених (зашифрованих) даних.
2.  **Динамічна зміна логера:** Перемикання LoggerManager на файловий логер без зміни бізнес-логіки.
3.  **Динамічна зміна стратегії:** Перемикання з шифрування на стиснення даних.

## Висновок
Інтеграція патернів дозволила створити систему з низькою зв'язаністю (Low Coupling). Кожен компонент відповідає за свою частину роботи, а їхня взаємодія через інтерфейси робить код легким для тестування та розширення.
