# Лабораторна робота №24: Strategy + Observer

## Опис реалізації
У проєкті реалізовано поєднання двох патернів для гнучкої обробки чисел:

1. **Strategy**: Дозволяє обчислювати квадрат, куб або корінь числа. Клас `NumericProcessor` не знає деталей обчислень, він лише делегує роботу обраній стратегії. Це дозволяє додавати нові операції без зміни коду процесора (OCP).
2. **Observer**: Реалізований через події (`event Action`). При кожному розрахунку об'єкт `ResultPublisher` сповіщає всіх підписаних спостерігачів.

### Спостерігачі:
- **ConsoleLogger**: Виводить дані в реальному часі.
- **HistoryLogger**: Накопичує список усіх виконаних операцій.
- **ThresholdNotifier**: Спрацьовує лише тоді, коли результат перевищує 100.

## Висновки
Поєднання цих патернів дозволяє будувати системи з низькою зв'язністю. Обробка даних (Strategy) повністю відділена від реакції на ці дані (Observer).
