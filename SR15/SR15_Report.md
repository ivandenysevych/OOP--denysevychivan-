# Звіт з аналізу SOLID принципів (SRP, OCP) в Open-Source проєкті

## 1. Обраний проєкт
- **Назва:** Entity Framework Core (EF Core)
- **Посилання на GitHub:** https://github.com/dotnet/efcore

## 2. Аналіз SRP (Single Responsibility Principle)

### 2.1. Приклади дотримання SRP
#### Клас: DbSet<TEntity>
- **Відповідальність:** Представлення колекції об'єктів у БД для виконання запитів.
- **Обґрунтування:** Клас не займається підключенням чи SQL логікою, лише інтерфейс для LINQ.

### 2.2. Приклади порушення SRP
#### Клас: DbContext
- **Множинні відповідальності:** Керування підключенням, Change Tracking, транзакції.
- **Проблеми:** "God Object", занадто багато причин для зміни класу.

## 3. Аналіз OCP (Open/Closed Principle)

### 3.1. Приклади дотримання OCP
#### Модуль: Провайдери баз даних
- **Механізм розширення:** Інтерфейс IDbContextOptionsExtension.
- **Обґрунтування:** Дозволяє додавати підтримку нових БД (наприклад, Oracle) без зміни ядра EF Core.

### 3.2. Приклади порушення OCP
#### Модуль: RelationalSqlTranslatingExpressionVisitor
- **Проблема:** Додавання нових SQL функцій часто вимагає модифікації існуючих методів VisitBinary.

## 4. Загальні висновки
Проєкт демонструє чудову модульність через DI, але має перевантажені базові класи через масштабність.
